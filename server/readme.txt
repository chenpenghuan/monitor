监控改进设计：
一、报警模块：
    1.用插件（如testplugin.py）来获取报警结果，并返回到server（如recieve.py）
    2.server拿到报警结果后决定发邮件、短信还是打电话通知，并将报警结果记录到报警记录数据库中
    3.前台配置报警时，php程序会将server的配置文件置空，当server收到报警结果，就会从数据库中读取配置并刷新配置文件


    改进效果：
        编写插件不用再重复编写发邮件、发短信、打电话和写入数据库的过程
二、统计模块：
    1.将第一个字段的字段名（cont_conf中cont_var）作为web表中的键，其他的字段中的字段名作为二级键，如exam文件夹中的程序所示
    2.在后台程序扫描时，按键取值得到行数据（数据类型为数组），把行数据遍历写入数据库。
